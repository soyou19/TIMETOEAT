## 一、项目概述

### 1.1 项目定位

**一句话定义**: 拍照识别处方单,自动生成服药提醒的鸿蒙原生应用

**核心价值**:
- 解决长期服药人群(老年人、慢性病患者)的服药提醒痛点
- 通过OCR技术降低手动输入成本
- 本地化处理保护用户隐私

### 1.2 目标用户

- **需要长期服药的慢性病患者**: 如高血压、糖尿病患者,需要每天定时服药
- **需要短期服药的急性病患者**: 如感冒、感染,需要按疗程服药
- **帮助父母管理服药的子女**: 为父母设置提醒,远程查看服药情况(未来功能)

### 1.3 核心功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 首次启动配置 | 引导用户设置用餐时间、作息时间、性别、年龄 | P0 |
| 拍照识别 | 拍摄处方单,自动识别药品信息 | P0 |
| 识别结果编辑 | 编辑确认识别结果,支持多页处方拼接 | P0 |
| 时间解析 | 解析医嘱文本,生成具体时间点 | P0 |
| 提醒通知 | 到点推送服药提醒,支持多药品合并提醒 | P0 |
| 服药记录 | 记录服药历史,生成服药日历 | P0 |
| 疗程管理 | 剩余药量倒计时,药快吃完提醒 | P1 |
| 导出功能 | 导出服药记录(PDF/Excel) | P1 |
| 数据管理 | 一键删除所有数据 | P1 |

### 1.4 功能边界

**做的**:
- ✅ 拍照识别机器打印的处方单
- ✅ 本地化OCR处理,不上传图片
- ✅ 自动识别药品名称、用法用量、频次
- ✅ 生成服药提醒,到点推送通知
- ✅ 记录服药历史,支持导出

**不做的**:
- ❌ 不支持手写处方识别
- ❌ 不涉及在线问诊
- ❌ 不卖药
- ❌ 不保存患者真实身份信息(姓名、身份证号等)
- ❌ 不做诊断建议

### 4.3 时间解析流程

```typescript
// 时间解析引擎核心逻辑

class TimeParser {
  /**
   * 解析医嘱文本,生成频次规则
   * @param frequencyText 医嘱文本, 如 "一日三次,饭后服用"
   * @param userProfile 用户配置
   * @returns 频次规则
   */
  parse(frequencyText: string, userProfile: UserProfile): FrequencyRule {
    // 步骤1: 识别频次类型
    const frequencyType = this.identifyFrequencyType(frequencyText);

    // 步骤2: 识别相对用餐时间
    const mealRelation = this.identifyMealRelation(frequencyText);

    // 步骤3: 计算具体时间点
    const timePoints = this.calculateTimePoints(
      frequencyType,
      mealRelation,
      userProfile
    );

    return {
      type: frequencyType.type,
      times: frequencyType.times,
      timePoints: timePoints,
      relativeToMeal: mealRelation.relativeToMeal,
      mealOffsetMinutes: mealRelation.offset
    };
  }

  /**
   * 识别频次类型
   */
  private identifyFrequencyType(text: string): { type: string; times: number } {
    // 规则匹配(按优先级排序)
    const rules = [
      { pattern: /一日三次|每天三次|每日三次/, type: 'daily', times: 3 },
      { pattern: /一日两次|每天两次|每日两次/, type: 'daily', times: 2 },
      { pattern: /一日一次|每天一次|每日一次/, type: 'daily', times: 1 },
      { pattern: /隔日一次|隔天一次/, type: 'interval', times: 1, intervalHours: 48 },
      { pattern: /每(\d+)小时一次/, type: 'interval', dynamic: true },
      { pattern: /必要时|需要时/, type: 'asNeeded', times: 1 },
      { pattern: /每周[一二三四五六日天]/, type: 'weekly', dynamic: true }
    ];

    for (const rule of rules) {
      const match = text.match(rule.pattern);
      if (match) {
        if (rule.dynamic) {
          return this.parseDynamicRule(text, rule, match);
        }
        return { type: rule.type, times: rule.times };
      }
    }

    // 默认返回一日一次
    return { type: 'daily', times: 1 };
  }

  /**
   * 识别相对用餐时间
   */
  private identifyMealRelation(text: string): { relativeToMeal: string; offset: number } {
    if (/饭前|餐前/.test(text)) {
      return { relativeToMeal: 'before', offset: -30 };
    }
    if (/饭后|餐后/.test(text)) {
      return { relativeToMeal: 'after', offset: 30 };
    }
    return { relativeToMeal: 'none', offset: 0 };
  }

  /**
   * 计算具体时间点
   */
  private calculateTimePoints(
    frequencyType: any,
    mealRelation: any,
    userProfile: UserProfile
  ): string[] {
    const timePoints: string[] = [];

    if (frequencyType.type === 'daily') {
      // 根据次数分配时间点
      const times = frequencyType.times;

      if (times === 1) {
        // 一日一次: 默认早上9点
        timePoints.push('09:00');
      } else if (times === 2) {
        // 一日两次: 早饭和晚饭后
        timePoints.push(
          this.addMinutes(userProfile.mealTimes.breakfast, mealRelation.offset),
          this.addMinutes(userProfile.mealTimes.dinner, mealRelation.offset)
        );
      } else if (times === 3) {
        // 一日三次: 三餐后
        timePoints.push(
          this.addMinutes(userProfile.mealTimes.breakfast, mealRelation.offset),
          this.addMinutes(userProfile.mealTimes.lunch, mealRelation.offset),
          this.addMinutes(userProfile.mealTimes.dinner, mealRelation.offset)
        );
      }
    }

    return timePoints;
  }

  /**
   * 时间加减计算
   */
  private addMinutes(time: string, minutes: number): string {
    const [hours, mins] = time.split(':').map(Number);
    const totalMinutes = hours * 60 + mins + minutes;
    const newHours = Math.floor(totalMinutes / 60) % 24;
    const newMins = totalMinutes % 60;
    return `${String(newHours).padStart(2, '0')}:${String(newMins).padStart(2, '0')}`;
  }
}
```

```typescript
function desensitize(text: string): {
  processedText: string;
  detectedInfo: SensitiveInfo[];
} {
  const detectedInfo: SensitiveInfo[] = [];
  let processedText = text;

  for (const [type, patterns] of Object.entries(SENSITIVE_PATTERNS)) {
    for (const pattern of patterns) {
      processedText = processedText.replace(pattern, (match, ...args) => {
        // 记录检测到的敏感信息
        detectedInfo.push({
          type: type as SensitiveInfoType,
          originalText: match,
          position: { start: 0, end: 0 }, // 实际实现时需要计算位置
          confidence: 0.9
        });

        // 根据类型进行脱敏
        switch (type) {
          case 'name':
            return '***';
          case 'idCard':
            return match.slice(0, 6) + '********' + match.slice(-1);
          case 'phone':
            return match.slice(0, 3) + '****' + match.slice(-4);
          case 'diagnosis':
            return '***';
          case 'address':
            return '***';
          default:
            return '***';
        }
      });
    }
  }

  return { processedText, detectedInfo };
}
```

```typescript
/**
 * 生成隐私提示信息
 */
function generatePrivacyNotice(detectedInfo: SensitiveInfo[]): PrivacyNotice {
  // 已保存的信息类型
  const savedInfo = [
    '药品名称',
    '用法用量',
    '提醒时间',
    '服药记录'
  ];

  // 已丢弃的信息类型(根据检测到的敏感信息)
  const discardedInfo = detectedInfo.map(info => {
    switch (info.type) {
      case 'name':
        return '患者姓名';
      case 'idCard':
        return '身份证号';
      case 'phone':
        return '手机号';
      case 'diagnosis':
        return '诊断信息';
      case 'address':
        return '地址';
      default:
        return '敏感信息';
    }
  });

  // 去重
  const uniqueDiscardedInfo = [...new Set(discardedInfo)];

  // 生成提示消息
  let message = '';
  if (uniqueDiscardedInfo.length > 0) {
    message = `已自动过滤${uniqueDiscardedInfo.join('、')}等敏感信息`;
  } else {
    message = '未检测到敏感信息';
  }

  return {
    savedInfo,
    discardedInfo: uniqueDiscardedInfo,
    message
  };
}
```

### 9.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| HMS ML Kit OCR准确率不足 | 高 | 中 | 1. 优化预处理逻辑<br>2. 提供手动编辑功能<br>3. 收集用户反馈持续优化 |
| 鸿蒙本地通知不稳定 | 高 | 低 | 1. 实现多重提醒机制<br>2. 提供应用内提醒<br>3. 用户可设置提前提醒 |
| 数据库性能问题 | 中 | 低 | 1. 合理设计索引<br>2. 定期清理历史数据<br>3. 实现数据归档功能 |

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
